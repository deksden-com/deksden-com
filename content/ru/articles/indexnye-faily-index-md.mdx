# Индексные файлы index.md для memory bank

В больших папках memory bank простой список файлов почти бесполезен: агент видит имена, но не понимает смысл каждого документа.

Поэтому я использую отдельный `index.md` в ключевых каталогах (`architecture/`, `guides/`, `epics/`) и держу в нем краткие описания файлов.

## Что хранить в index.md

- список файлов папки;
- описание каждого файла в 1-2 предложения;
- описание берем из frontmatter поля `description`;
- для сложных деревьев можно делать обзор на несколько уровней вниз.

## Почему это работает

Без индекса агенту приходится:

- читать много файлов подряд;
- или угадывать по именам;
- или искать ключевые слова с риском пропустить нужное.

С индексом агент быстрее решает, что читать, а что пропустить. Это экономит токены, время и контекст оркестратора.

## Как обновлять индекс без ручной боли

Практичный вариант:

1. Выделить вспомогательного агента для файловой системы.
2. Запускать обновление по крупным веткам memory bank параллельно.
3. На каждом файле агент обновляет `description`, после чего пересобирает `index.md`.
4. Для специальных директорий задать исключения и отдельные правила.

Такой процесс лучше обычного скрипта, когда нужно не механически переписать дерево, а аккуратно пересобрать осмысленные описания.

## Итог

`index.md` — простой инструмент, который заметно улучшает навигацию агентов в проектных знаниях.
